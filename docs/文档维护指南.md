# 文档维护指南

## 📁 文档结构

```
docs/
├── README.md          # 主文档（需求说明 + 使用指南）
└── TODO.md           # 实现计划和进度
```

## 🔄 更新规则

### 重要原则

⚠️ **所有需求变更都必须更新 `docs/README.md`**

### 何时更新

当你需要：

1. ✅ **添加新功能** - 更新需求说明和技术架构
2. ✅ **修改现有功能** - 更新相关章节
3. ✅ **修复问题** - 更新常见问题章节
4. ✅ **添加 API** - 更新 API 接口章节
5. ✅ **配置变更** - 更新配置说明章节

### 更新步骤

#### 1. 更新 docs/README.md

根据变更内容，更新相应章节：

- 新功能 → 项目概述、需求说明
- 架构调整 → 技术架构
- 新 API → API 接口
- 配置修改 → 配置说明
- 使用变化 → 快速开始、访问指南

#### 2. 更新 docs/TODO.md

- 添加新任务到 TODO 列表
- 标记已完成任务为 `[x]`
- 更新实现进度

#### 3. 验证文档

```bash
# 检查文档格式
cat docs/README.md | head -50

# 验证链接
grep -E "^\[.*\]\(.*\)" docs/README.md
```

## 📝 文档模板

### 添加新 API

在 `API 接口` 章节添加：

```markdown
### X. 功能名称

**接口**: `POST /api/xxx`

**请求**:
```json
{
  "param": "value"
}
```

**响应**:
```json
{
  "result": "success"
}
```
```

### 添加常见问题

在 `常见问题` 章节添加：

```markdown
### X. 问题描述

**原因**: 问题产生的原因

**解决**:
```bash
# 解决命令
command
```
```

### 添加配置项

在 `配置说明` 章节添加：

```markdown
#### [section] - 配置说明

```ini
key = value    # 配置说明
```
```

## 🎯 最佳实践

### ✅ 推荐

- 保持文档与代码同步
- 使用清晰的标题和结构
- 提供完整的示例代码
- 包含错误处理和解决方案
- 定期审查和更新文档

### ❌ 避免

- 文档与实际功能不符
- 缺少必要的配置说明
- 没有错误处理说明
- 过时的使用示例
- 断裂的链接

## 📖 文档章节说明

### docs/README.md

包含以下章节：

1. **项目概述** - 项目简介和特性
2. **需求说明** - 工作流程和 RAG 功能
3. **技术架构** - 整体架构和技术栈
4. **快速开始** - 安装和启动步骤
5. **访问指南** - Web 界面和测试方法
6. **配置说明** - 所有配置项详解
7. **API 接口** - 所有 API 接口文档
8. **常见问题** - 问题和解决方案

### docs/TODO.md

包含：

- 文档维护说明
- 实现计划列表
- 任务进度跟踪

## 🔍 检查清单

每次更新文档前，确认：

- [ ] 已更新 `docs/README.md` 相关章节
- [ ] 已更新 `docs/TODO.md` 进度
- [ ] 文档格式正确
- [ ] 示例代码可运行
- [ ] 配置说明完整
- [ ] 链接有效
- [ ] 语言清晰准确
