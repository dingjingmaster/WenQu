## 📁 文档结构

```
docs/
├── README.md          # 主文档（需求说明 + 使用指南）
├── TODO.md           # 实现计划和进度

## 🔄 更新规则

### 重要原则

⚠️ **所有需求变更都必须更新 `docs/README.md`**

### 何时更新

当你需要：

1. ✅ **添加新功能** - 更新需求说明和技术架构
2. ✅ **修改现有功能** - 更新相关章节
3. ✅ **修复问题** - 更新常见问题章节
4. ✅ **添加 API** - 更新 API 接口章节
5. ✅ **配置变更** - 更新配置说明章节

### 更新步骤

#### 1. 更新 docs/README.md

根据变更内容，更新相应章节：

- 新功能 → 项目概述、需求说明
- 架构调整 → 技术架构
- 新 API → API 接口
- 配置修改 → 配置说明
- 使用变化 → 快速开始、访问指南

#### 2. 更新 docs/TODO.md

- 添加新任务到 TODO 列表
- 标记已完成任务为 `[x]`
- 更新实现进度

#### 3. 验证文档

```bash
# 检查文档格式
cat docs/README.md | head -50

# 验证链接
grep -E "^\[.*\]\(.*\)" docs/README.md
```

## 🎨 UI 开发文档

### 前端项目结构

```
src/ui/
├── index.html         # HTML 入口文件
├── package.json       # npm 依赖配置
├── vite.config.js     # Vite 构建配置
├── src/
│   ├── main.js        # Vue 入口文件
│   ├── App.vue        # 根组件
│   ├── components/    # Vue 组件
│   │   └── MainView.vue  # 主界面组件
│   ├── assets/        # 静态资源
│   └── styles/        # 样式文件
│       └── main.css   # 全局样式
└── dist/              # 构建输出目录
```

### UI 技术栈

- **框架**: Vue 3 (Composition API)
- **UI 库**: Element Plus
- **动画库**: Pretext.js (CDN 引入)
- **构建工具**: Vite 5
- **Markdown 渲染**: marked
- **代码高亮**: highlight.js

### 前端开发流程

1. **开发模式**:
```bash
cd src/ui
npm run dev
```

2. **构建生产版本**:
```bash
cd src/ui
npm run build
```

3. **集成到服务器**:
```bash
cp -r dist/* ../server/static/
```

### UI 功能说明

#### 主要组件

- **MainView.vue**: 主界面组件
  - Logo 展示（Google 风格彩色设计）
  - 用户输入框（支持多行文本）
  - 文件/文件夹上传
  - 执行按钮
  - WebSocket 实时通信
  - 执行过程展示（可折叠）
  - 结果展示（Markdown 渲染，可折叠）

#### WebSocket 通信

- 连接端点：`/ws/agent`
- 消息类型：
  - `connected`: 连接成功
  - `log`: 日志信息
  - `progress`: 进度更新
  - `result`: 最终结果
  - `error`: 错误信息
  - `heartbeat`: 心跳包

#### API 接口

- `POST /api/agent/execute`: 执行 Agent 任务
  - 参数：`query` (表单), `files` (文件列表)
  - 响应：`{ success: boolean, message: string }`
