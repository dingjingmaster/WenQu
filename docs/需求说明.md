# WenQu需求说明

## 需求描述

WenQu是一个本地Agent，用于处理用户的问题和请求。其工作思路如下：
- 解析用户请求，根据用户请求 + 相应的提示词给到语言模型qwen，做进一步需求拆分，明确用户描述的所有关键点
- 基于上一步的分析结果做进一步分析，确定要实现用户需求需要做哪些步骤的工作并列成TODO列表
- 思考TODO列表是否可以满足用户的实际需求，如果不能满足则做进一步分析，如果觉得用户需求有不够清晰的地方则通过web界面与用户沟通，供用户输入更详细的需求描述，确认用户需求，直到用户需求清晰为止
- 根据上一步形成的TODO列表开始逐步实现用户需求，且每次实现一个TODO项后检查结果是否符合此项TODO规划时候的预期，如果不符合预期则做简单调整，直到满足预期则继续实现下一个TODO项，循环以上步骤，直到所有TODO项都被实现为止
- 最后在实现完成后，你需要思考并检查是否满足用户需求，如果不满足则检查哪些点不满足，并作进一步调整，直到满足为止则输出结果给到用户

为了满足上述工作思路，需要扩展Agent功能，为此增加 RAG 功能，具体思路如下：
- 基于用户提供的文档(可能是网页、PDF、txt、markdown格式)，使用embedding模型做语义分块，将文档中的文本内容转换为向量表示
- 基于上一步的向量表示，使用向量数据库 postgreSQL 存储文档向量
- 基于用户请求，使用向量数据库 postgreSQL 做相似度搜索，返回与用户请求最相关的文档向量
- 基于上一步的返回结果，使用语言模型qwen生成最终的回复结果

## 需要的技术框架

- UI层：使用 Open WebUI，可以按需修改，整合到python实现的服务器中
- Agent层：使用 LangChain 框架
- RAG层：使用 postgreSQL 向量数据库，存储向量化数据，使用本地的llama-server提供的qwen3-embedding-8b实现向量化+reranker(bge-reranker)召回模型，这块使用LlamaIndex框架
- 推理层：使用本地的llama-cpp

整个系统功能尽可能都用python实现，且使用python包管理器uv管理依赖
