# WenQu 访问指南

## 🌐 访问网页

启动服务器后，在浏览器中访问：

### 主页
```
http://localhost:8000
```

这将显示一个漂亮的 Web 界面，可以：
- 📝 输入你的需求
- 🚀 点击"分析需求"按钮
- 📋 查看 AI 生成的 TODO 列表

### API 文档
```
http://localhost:8000/docs
```

Swagger UI 界面，可以测试所有 API 接口

### 健康检查
```
http://localhost:8000/health
```

返回：`{"status": "healthy"}`

## 📱 网页功能

### 1. 需求分析
在主页的文本框中输入你的需求，例如：

```
帮我创建一个简单的 Python Web 应用，需要实现用户登录功能
```

点击"🚀 分析需求"按钮，系统会：
- 🎯 分析关键需求
- ❓ 指出不清晰点
- 📝 生成 TODO 列表

### 2. 查看结果
分析结果会以卡片形式展示：
- 关键需求列表
- 不清晰点列表
- TODO 任务列表

## ⚠️ 常见问题

### 1. 页面空白
**原因**: 服务器未启动或启动失败

**解决**:
```bash
# 检查服务器是否运行
ps aux | grep uvicorn

# 重新启动服务器
./start.sh
```

### 2. 无法连接
**原因**: 端口被占用或防火墙阻止

**解决**:
```bash
# 检查端口占用
lsof -ti:8000 | xargs kill -9

# 修改配置文件使用其他端口
# 编辑 config/config.ini
[server]
port = 8001
```

### 3. API 返回错误
**原因**: llama-server 未运行或配置错误

**解决**:
- 确保 llama-server 已启动
- 检查 config/config.ini 中的 base_url
- 查看服务器日志获取详细错误信息

## 📖 API 使用示例

### 分析需求
```bash
curl -X POST http://localhost:8000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "user_input": "帮我创建一个 Python Web 应用"
  }'
```

### 执行任务
```bash
curl -X POST http://localhost:8000/api/execute \
  -H "Content-Type: application/json" \
  -d '{
    "task_content": "创建 Hello World 应用",
    "context": "FastAPI 示例"
  }'
```
